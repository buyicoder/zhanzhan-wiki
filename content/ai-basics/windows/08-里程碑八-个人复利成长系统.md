> **阶段目标：** 把课程里练出的所有能力，组装成一台属于你自己的「复利机器」：你的每一天输入都变成可积累、可复用的资产。
> **阶段成功标志：** 拥有一个持续更新的个人资产库（GitHub）、一个记录-复盘习惯（日记仓库）、以及一个「认识你」的 AI 第二外脑。
> **⏱️ 预计总用时：** 初始搭建 2 小时，之后每天 10–20 分钟

## 想清楚一件事：课程结束后，拉开差距的是什么

同样上完这门课的两个人，一年后差距会非常大。差别不在谁更聪明，而在——

**一个人用完 AI 就散场；另一个人把每次使用都变成资产。**

回想你这门课积累了什么：几个 GitHub 仓库、一个上线的游戏、一堆和 AI 的对话、无数次踩坑和解决。如果它们散落各处，一个月后就蒸发了；如果它们被系统地记录和沉淀，它们会**复利**——你的 AI 会越来越懂你，你的下一个项目会站在上一个的肩膀上。

这就是复利循环：

```text
记录 → 总结 → 沉淀 → 复用 → （更强的你）→ 记录 → ……
```

这一章教你把这个循环跑起来。

## 任务 1：把 GitHub 变成你的个人资产库

🎯 **成功标志：** 你的 GitHub 主页有一份个人 README；本课所有项目都已推送；从今天起「所有项目都上 GitHub」成为默认动作。
⏱️ 预计用时：30 分钟

**操作步骤**

1. 检查资产：本课的 `ai-beginner-practice`、`file-organizer`、`snake-game` 是否都在 GitHub 上。
2. 创建一个和你用户名同名的仓库（如用户名是 `zhangsan`，仓库就叫 `zhangsan`）——GitHub 会把它的 README 展示在你的个人主页上。
3. 让 Agent 帮你写主页 README：

   ```text
   帮我写一份 GitHub 个人主页 README：我是谁、我正在学什么、我做过的项目（贪吃蛇游戏+网址、文件整理工具），语气真诚，不吹嘘。
   ```

   📸 截图 M8-1-1：完成后的 GitHub 个人主页

🔍 **为什么这件事重要**
你的 GitHub 从此有两重身份：对外是**名片**（Show me your code），对内是**记忆**——你做过的每件事都可检索、可恢复、可复用。里程碑五那句话现在兑现了：每一个项目都是你的个人资产。

## 任务 2：建立日记仓库——最值钱的数据是你对自己的复盘

🎯 **成功标志：** 存在一个私有仓库 `my-journal`，里面至少有 3 篇日记，并且你连续 3 天完成了记录。
⏱️ 预计用时：初建 20 分钟，每天 10 分钟

**操作步骤**

1. 创建**私有**仓库 `my-journal`，clone 到本地。
2. 让 Agent 建立结构：

   ```text
   在这个文件夹里建立日记仓库结构：按 年/月 分文件夹，每天一个 Markdown 文件。再给我一个日记模板，包含：今天做了什么、遇到什么问题怎么解决的、学到了什么、明天最重要的一件事。
   ```

3. 今天就写第一篇。写完走固定动作：`git add . && git commit -m "日记" && git push`。
4. 连续坚持 3 天。

> 💡 写什么不重要，**写下你的真实想法**才重要。尤其是：做了什么决定、为什么、卡在哪、怎么想通的。这类「元认知」内容是了解你的第一手资料——比任何简历都真实。
>
> 💡 人生发生重大转折的时候，务必写一篇长的。几年后你会感谢自己。

🔬 **深入一层**
为什么用 Git 管日记而不是备忘录？三个理由：历史全部保留（你能看到自己想法的演变）；纯文本 AI 可读（下一个任务就用上）；换电脑永不丢失。你在用工程师的基础设施管理自己的人生数据。

## 任务 3：让 AI 第一次「认识你」

🎯 **成功标志：** AI 基于你的真实数据（GitHub + 日记）给出一份「你是谁」的画像，而且你觉得**准**。
⏱️ 预计用时：30 分钟

**操作步骤**

1. 在你的日记仓库文件夹启动 Claude（或打开 WorkBuddy 指向该文件夹）。
2. 输入：

   ```text
   请阅读这个文件夹里我所有的日记，再看看我的 GitHub 主页（<你的主页网址>）上的项目。然后告诉我：
   1. 你觉得我是一个什么样的人？
   2. 我最近真正关心的问题是什么？
   3. 你观察到我有什么做事模式（好的和不好的）？
   4. 基于以上，你建议我下一步学什么、做什么？
   ```

3. 认真读它的回答。对的地方，你会起鸡皮疙瘩；不对的地方，纠正它——然后把纠正也记进今天的日记。

🔍 **刚才发生了什么**
这是全课最重要的一次范式转换：前七个里程碑，你在**使用** AI（它对谁都一样）；从这一刻起，你在**喂养**一个认识你的 AI（它对你独一无二）。数据越多，它越懂你。而这些数据你已经在自然地生产了——只要你保持记录的习惯。

> 💡 还有更多数据可以喂：你和豆包/DeepSeek 的历史对话可以导出，你的旧笔记可以整理进仓库。留下的数据资产越多，AI 对你的了解越立体。

## 任务 4：搭建每日复利循环

🎯 **成功标志：** 连续 7 天完成「晨间三件事 + 晚间复盘」，并且至少一次明显感到「AI 记得我之前说过的话」。
⏱️ 每天 15 分钟

**操作步骤**

每天早上（5 分钟），在你的日记/工作区文件夹对 AI 说：

```text
读一下我最近三天的日记。基于我正在做的事，帮我定今天最重要的三件事：一件推进主线，一件建设系统，一件照顾状态。
```

每天晚上（10 分钟）：

```text
我今天做了：<简单罗列>。帮我复盘：哪些真正推进了目标？哪些只是看起来忙？明天的三件事建议是什么？把复盘写成今天的日记，给我确认后保存。
```

确认内容没问题后让它保存，然后 `git push`。

🔍 **为什么这个循环是「复利」的**
每天的复盘会成为明天决策的输入；每周的日记会让 AI 的建议越来越贴合你；每个项目的经验会被下一个项目复用。单利是「用完就散场」，复利是**今天的输出成为明天的本金**。坚持一个月，回头看第一周的日记，你会看到肉眼可见的成长曲线。

🔬 **深入一层：这套系统的完全体**
你现在手工跑的这个循环，可以被系统化成一个真正的「第二外脑」：自动扫描你的每日活动、分层管理记忆（原始记录 → 提炼 → 长期资产）、早晚自动给你做规划和复盘、甚至替你给不同的 AI 分派任务再验收报告。这就是「占占 AI」体系的下一阶段——**Jarvis 个人 AI 幕僚长系统**，也是进阶课程的终点站。你在本课建立的日记仓库和记录习惯，就是它的地基：到时候你的 Jarvis 入职的第一天，读完你的日记仓库，它就已经认识你了。

## 任务 5（毕业仪式）：写给一年后的自己

🎯 **成功标志：** 日记仓库里有一篇《写给一年后的我》，并已 push。

写下三件事：

1. 上这门课之前，你对 AI 的恐惧或误解是什么？现在呢？
2. 你的贪吃蛇网址（一年后点开它，看看还在不在——它会在的）。
3. 一年后你希望你的 GitHub 上多出什么？你的 AI 到那时应该有多懂你？

写完，commit 信息就写：`毕业`。
