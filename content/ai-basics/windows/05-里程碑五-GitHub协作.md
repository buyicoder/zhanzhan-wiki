## 这一关解决什么问题

只存在自己电脑里的项目很难协作，也无法证明过程。GitHub 的价值不只是“云盘”，而是把需求、修改、审查和版本放进同一条可追踪链路。

## 你将完成什么

- 把一个本地项目保存到 GitHub，并在另一处重新取得它。
- 用 Issue 写清一个需求，再在独立分支完成修改。
- 发起 Pull Request，检查差异后合并，留下完整协作记录。

## 成功标准

仓库可以从空文件夹重新取得并运行；另一个人只看 Issue、提交和 PR，也能知道需求是什么、改了什么、如何验收。

## 能力迁移

你开始把项目当数字资产，而不是聊天窗口里的临时输出。以后与人或 Agent 协作，都可以围绕异步产物而不是反复口头交代。
