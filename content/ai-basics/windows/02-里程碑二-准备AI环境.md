> **阶段目标：** 搭好一套完整的 AI 工具链，让你既能用国内工具日常干活，也能用上全球最先进的 Agent。
> **阶段成功标志：** 能访问国内外 AI 资源，并至少成功使用 WorkBuddy、Codex、Claude + DeepSeek 三种 Agent 环境。
> **⏱️ 预计总用时：** 90–120 分钟（多数时间在等待和注册）

## 本章地图

```text
2.1 全球互联网环境   →  能访问 chatgpt.com 等全球网站
2.2 Codex           →  用上 OpenAI 的顶级 Agent
2.3 Claude+DeepSeek →  最强 Agent 外壳 + 国产模型引擎，不开代理也能用
```

> 💡 为什么要三套环境：WorkBuddy 是无门槛的日常主力；Codex 和 Claude Code 是全球最先进的 Agent，能力上限更高。学会「魔法」（访问全球互联网）能极大提高你的能力上限——第一手资讯、更多工具、更广视野。但如果暂时不想折腾，只用国内工具也可以继续学习，本章 2.1 和 2.2 可以之后再回来补。

## 2.1 全球互联网环境

### 任务 1：配置网络客户端

🎯 **成功标志：** 客户端显示已导入的订阅，至少一个节点显示健康的延迟数字。
⏱️ 预计用时：30 分钟
🔗 快捷网址：主服务 https://congyu.moe ｜ 备用服务 https://www.miyun.co

**操作步骤**

1. 打开主服务官网（上方链接）。
   > 📸 操作截图尚未补齐：官网首页
2. 用你常用的邮箱注册并登录。
3. 进入「商店」，选择「几乎不限时流量包」，付款购买。
   > 📸 操作截图尚未补齐：商店套餐页，目标套餐圈红
4. 等待开通期间，找到「工具下载」，选择 Windows 下载客户端（Clash 类工具）并安装。
   > 📸 操作截图尚未补齐：工具下载页
5. 回到首页，找到你的订阅链接，点击「一键导入到 Clash」。
   > 📸 操作截图尚未补齐：一键导入按钮位置
6. 打开客户端，确认订阅已导入、节点列表有延迟数字显示。
   > 📸 操作截图尚未补齐：客户端节点列表，健康延迟圈绿
🔍 **刚才发生了什么**
「订阅」是服务商给你的节点清单，客户端按这份清单把你的网络流量转发到海外节点，从而访问全球网站。开关客户端 = 开关「魔法」。

### 任务 2：验证全球访问

🎯 **成功标志：** chatgpt.com、youtube.com、x.com 三个网站都能加载出主页面，而不是网络错误页。

**操作步骤**

1. 确认客户端开启、节点已选中。
2. 依次访问：https://chatgpt.com → https://youtube.com → https://x.com
   > 📸 操作截图尚未补齐：三个网站加载成功的浏览器标签
> ⚠️ 打不开怎么办（按顺序试）：
> 1. 客户端里点「刷新/测速」；
> 2. 切换一个延迟更低的节点；
> 3. 都不行 → 换备用服务（觅云，流程同任务 1，选最低档套餐即可）。

### 任务 3：练习开关网络（必练）

🎯 **成功标志：** 能主动切换一次节点；退出客户端后，普通国内网站仍然可以正常打开。

**操作步骤**

1. 在客户端中手动切换到另一个节点，刷新网页确认仍能访问。
2. 完全退出客户端，访问一个国内网站（如 baidu.com），确认正常。
3. 重新打开客户端，确认「魔法」恢复。

> ⚠️ 最常见的事故：退出客户端后**所有网站都打不开**。原因是系统代理没有被正确关闭。解决：系统设置 → 网络 → 详细信息 → 代理，把打开的代理开关全部关掉。这个操作请现在就练一遍，记住位置。

🔬 **深入一层**
你刚才操作的「代理」是一种网络中转：浏览器 → 本机客户端 → 海外节点 → 目标网站。理解了这个链条，你就明白为什么客户端退出时要恢复系统设置，也明白了为什么里程碑 2.3 的国产模型方案不需要这一切。

## 2.2 Codex（OpenAI 顶级 Agent）

### 任务 4：安装并登录 Codex

🎯 **成功标志：** Codex 能新建任务，发送「你好」后收到正常回答。
⏱️ 预计用时：30 分钟
🔗 快捷网址：套餐充值教程 https://getgpt.pro

**操作步骤**

1. 打开「魔法」。
2. 在 WorkBuddy 中输入：

   ```text
   帮我下载codex UI版
   ```

3. 等待安装完成，打开 Codex。
   📸 截图 M2-4-1：Codex 首次打开界面
4. 打开 OpenAI 官网登录账号（没有就注册一个）。
5. 参照 https://getgpt.pro 的教程完成套餐充值。
   📸 截图 M2-4-2：套餐生效后的账户状态
6. 回到 Codex，新建任务，发送「你好」，确认有回答。

> 💡 这一步体现了本课的元技能：**用手上已有的 Agent（WorkBuddy）去安装下一个更强的 Agent（Codex）。** 以后你的工具箱就是这样滚雪球的。

### 任务 5：用 Codex 安装 Claude CLI

🎯 **成功标志：** 终端运行 `claude --version` 能输出版本号。

**操作步骤**

1. 在 Codex 中输入：

   ```text
   帮我下载claude CLI
   ```

2. 完成后，打开终端（右键开始按钮 → 终端；或按 Win 键搜索「PowerShell」），输入：

   ```bash
   claude --version
   ```

   📸 截图 M2-5-1：终端显示 claude 版本号

🔍 **刚才发生了什么**
CLI 的意思是「命令行界面」——没有窗口按钮、在终端里用文字交互的程序。Claude Code 就是一个 CLI 形态的 Agent，它是当前全球最强的编程 Agent 之一。里程碑三会正式教你命令行，现在你只需要确认它装上了。

## 2.3 Claude + DeepSeek（日常主力配置）

> 这套组合的妙处：Claude Code 是最强的 Agent「外壳」，DeepSeek 是便宜好用的国产「引擎」。组合之后——**不开魔法也能用**，这就是你以后关掉代理也能干活的日常配置。

### 任务 6：安装 CC Switch 并配置 DeepSeek

🎯 **成功标志：** CC Switch 供应商列表出现 DeepSeek 配置，API Key 不再显示完整明文。
⏱️ 预计用时：20 分钟
🔗 快捷网址：DeepSeek 开放平台 https://platform.deepseek.com

**操作步骤**

1. 在 Codex（或 WorkBuddy）中输入：

   ```text
   帮我下载 CC Switch
   ```

2. 等待安装时，打开 DeepSeek 开放平台（上方链接），注册并充值 50 元。
   📸 截图 M2-6-1：DeepSeek 开放平台充值页
3. 左侧边栏选择「API keys」→ 点击「创建 API key」→ 命名为 `claude code` → 确定。
4. **立刻复制**显示出来的密钥（它只显示这一次）。
   📸 截图 M2-6-2：API key 创建弹窗（密钥部分打码）
5. 打开 CC Switch，点右上角 `+` 号：供应商选择 DeepSeek，API key 粘贴刚才的密钥，其它设置不动，点添加。
   📸 截图 M2-6-3：CC Switch 添加供应商界面

> ⚠️ API Key 就是你的「钱包钥匙」：谁拿到它谁就能花你的额度。不要发到群里，不要写进文件传给别人。忘了或泄露了就回平台删掉重建一个。

### 任务 7：在课程文件夹启动 Claude

🎯 **成功标志：** `pwd` 显示课程文件夹路径；输入 `claude` 后进入对话界面，界面显示当前使用 DeepSeek。

**操作步骤**

1. 打开终端（右键开始按钮 → 终端（或 Win 键搜索「PowerShell」））。
2. 输入以下命令进入课程文件夹：

   ```bash
   cd ~/Desktop/占占AI小白入门
   pwd
   ```

   确认输出以 `占占AI小白入门` 结尾。
3. 输入：

   ```bash
   claude
   ```

   📸 截图 M2-7-1：Claude Code 启动画面（可见 DeepSeek 字样）
4. 输入「你好」，确认有回答。

> ⚠️ `cd` 命令失败？最常见是文件夹名打错或路径不对。偷懒办法：输入 `cd `（注意有个空格），然后把文件夹从文件资源管理器直接**拖进终端窗口**，路径会自动填上，回车即可。再不行，把报错发给 WorkBuddy 问。

### 任务 8：验证「不开魔法也能用」

🎯 **成功标志：** 开启和关闭代理两种状态下，输入「你好」都能收到回答。

**操作步骤**

1. 保持 Claude 对话界面，先在「魔法」开启状态下发送「你好」。
2. 完全退出网络客户端（关闭魔法）。
3. 再发送一次「你好」，确认仍有回答。

🔍 **刚才发生了什么**
Claude Code 本体在你的电脑上运行，它把请求发给 CC Switch 配置的 DeepSeek 国内服务器——全程不出国门，所以不需要代理。**外壳（Agent 能力）和引擎（模型）是可以分开选的**，这是 AI 工具链最重要的一条认知。
