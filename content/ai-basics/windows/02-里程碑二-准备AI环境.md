## 这一关解决什么问题

第一次让 Agent 操作电脑之后，真正的障碍往往不是“不会问”，而是工具链不稳定：今天能用，明天登录失败；装了很多工具，却不知道哪个负责什么。这一关要建立的是可持续环境，而不是收集软件。

## 你将完成什么

- 分清日常对话工具、编辑器内 Agent 与能操作项目的执行型 Agent。
- 建立一套至少包含日常入口与复杂执行入口的组合。
- 知道遇到账号、网络、模型接入或终端问题时应该停在哪一层排查。

## 成功标准

你能在自己的 Windows 电脑上稳定打开至少一个日常 AI 工具和一个执行型 Agent，并用同一个小任务验证两者的分工；关闭再重新打开后，仍能继续工作。

## 能力迁移

以后换模型、换 Agent 或换电脑，你不必重新背一套软件名单。你会先判断任务需要“回答”还是“执行”，再检查接入、权限和工作目录三层条件。
