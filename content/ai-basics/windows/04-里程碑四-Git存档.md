## 这一关解决什么问题

Agent 改得越快，失控时破坏得也越快。没有版本存档，你只能祈祷它别改坏；有 Git，才敢让它大胆试错。

## 你将完成什么

- 为练习项目建立 Git 仓库和第一个可辨认的存档。
- 在 Agent 修改前后检查差异，而不是只听它口头汇报。
- 亲手恢复一次错误修改，并在分支里做一次隔离实验。

## 成功标准

你能在修改前确认工作区状态，修改后看懂差异，创建说明清楚的提交；项目被故意改坏后，可以恢复到上一个可用状态。

## 能力迁移

这套“先看状态、再改、再检查、再存档”的动作可以迁移到网站、文档、数据脚本和任何 Agent 项目。Git 不是程序员身份标志，而是后悔权。
