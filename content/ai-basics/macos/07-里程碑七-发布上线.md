> **阶段目标：** 完成从「作品」到「产品」的惊险一跃：拿到一个网址，让世界上任何人点开就能玩你的游戏。
> **阶段成功标志：** 获得一个公开网址；关闭本地项目后，用手机流量仍能打开贪吃蛇；并邀请一位真实用户试玩。
> **⏱️ 预计总用时：** 60–90 分钟

## 为什么还差一步

到这里，你已经完成了自己的第一个软件。但它还只能在你的电脑上运行。把项目文件发给朋友，对方要下载、找到 `index.html`、再用浏览器打开——对不懂电脑的人来说太复杂了。

真正的网站是这样的：你只发一个网址，对方点开就能用。例如：

```text
https://你的名字.github.io/snake-game/
```

我们缺的就是最后一环：

```text
本地项目 → 上传到 GitHub → 部署到互联网 → 获得网址 → 任何人都可以访问
```

---

## 任务 1：确认项目完整推送

🎯 **成功标志：** 本地 `git status` 是 clean；GitHub 上 `main` 的最新 commit 和本地 `git log -1` 一致。

```bash
git status
git log -1
git push
```

到 GitHub 网页核对最新 commit。

## 任务 2：完成第一次部署

🎯 **成功标志：** 托管平台显示部署成功（不是 building 或 failed），并生成网站链接。
⏱️ 预计用时：30 分钟

**操作步骤**

1. 问你的 Agent：

   ```text
   我有一个纯 HTML+CSS+JS 的贪吃蛇项目，已经推送到 GitHub。请给我推荐最简单的免费静态网站托管方案，并一步步指导我完成部署。我是初学者。
   ```

2. 按 Agent 的指导操作（最常见路径是 GitHub Pages：仓库 Settings → Pages → 选择 main 分支 → Save）。
   📸 截图 M7-2-1：托管平台部署成功的状态页
3. 等待部署状态变为成功，记下生成的网址。

🔍 **刚才发生了什么**
「部署」= 把你的代码放到一台 24 小时开机的电脑（服务器）上，由它对全世界提供访问。你自己的电脑可以关机了——从此这个游戏的「家」在云端。

> ⚠️ 部署后网页 404 或空白：最常见原因是入口文件不叫 `index.html`，或部署目录选错。把报错截图发给 Agent 排查。

## 任务 3：验证公开网址

🎯 **成功标志：** 无痕窗口（不登录任何账号）能打开网址，地址栏以 `https://` 开头。

1. 复制网址，开一个无痕窗口（Cmd + Shift + N）粘贴访问。
2. 确认地址栏是 `https://` 开头。
   📸 截图 M7-3-1：无痕窗口中运行的游戏（地址栏可见）

🔬 **深入一层**
`https` 里的 `s` 是加密的意思：访问者和服务器之间的数据是加密传输的，别人截获也读不懂。现代浏览器对非 https 网站会标「不安全」。你的托管平台自动帮你配好了这件事——但知道它的存在，是网站主人的基本素养。

## 任务 4：终极检验——关掉电脑照样能玩

🎯 **成功标志：** 电脑本地项目已关闭、手机 Wi-Fi 已关闭，用手机流量打开网址并完成一轮游戏。

1. 关闭电脑上所有本地服务和页面。
2. 手机关掉 Wi-Fi，只用流量。
3. 打开网址，完整玩一轮。
   📸 截图 M7-4-1：手机流量下运行的游戏

## 任务 5：邀请第一位真实用户

🎯 **成功标志：** 对方在自己的设备上打开并完成一轮游戏，全程没有接收任何项目文件。

把网址发给一位朋友，只发网址，不做任何解释——观察他能不能玩起来。

## 任务 6：收集反馈，完成第一次产品迭代

🎯 **成功标志：** GitHub 里有一条来自真实试玩的 Issue；有对应的修复 commit；新部署中问题已消失。

1. 朋友玩完后，问他一个问题：「哪里让你卡住或觉得别扭？」
2. 把它写成一条 Issue（这次的需求方是真实用户，不是你自己）。
3. 让 Agent 修复 → 亲自验证 → 提交推送 → 等待自动重新部署 → 在线上确认问题消失。
4. 回去关闭 Issue，并@朋友再试一次。

🔍 **刚才发生了什么——这一刻值得记住**
你刚刚完成了一个完整的产品循环：**发布 → 真实用户 → 反馈 → 修复 → 再发布**。全世界的互联网产品，不管多大，跑的都是这个循环。你已经用一个贪吃蛇把它走通了。

---

## 通往下一阶段的问题清单

发布之后，你很快会遇到新问题——这些正是进阶课的内容：

- 为什么修改代码后，网站不会立刻变化？（部署流程）
- 什么是服务器？什么是域名？
- 网站怎样保存用户的数据？（数据库）
- 怎样让用户注册登录？（账号系统）
- 怎样知道有多少人访问过？（数据统计）
- 网站故障后怎样恢复？（运维）
- 怎样把一个作品变成可以长期使用的产品？

**进阶课的最终任务：** 做出一个拥有网址、域名、HTTPS、账号系统和数据库的网站，邀请真实用户使用，并根据反馈完成第一次产品更新。

> 在入门课里，你学会了指挥 AI 写程序；在进阶课里，你要学会指挥 AI 交付产品。程序能够运行，只代表开发完成；**别人能够访问、理解、使用并愿意再次回来，才代表产品真正开始。**
