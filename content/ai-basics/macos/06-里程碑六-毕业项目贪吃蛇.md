> **阶段目标：** 综合运用前五个里程碑的全部能力，从零指挥 AI 完成一个真正能玩的游戏。
> **毕业成功标志：** 贪吃蛇可以启动、移动、转向、吃食物、变长、计分、判定失败、重新开始；GitHub 保留完整开发过程；换一个文件夹重新 clone 后仍然能玩。
> **⏱️ 预计总用时：** 4–6 小时（可分多天完成，每个阶段都是安全的存档点）

## 开工之前

**技术选型：** 浏览器版 `HTML + CSS + JavaScript`。不需要安装任何游戏库，双击 `index.html` 就能看到成果，成就感来得最快。

**工作节奏：** 每个阶段都走一遍里程碑五的完整循环——

```text
创建 Issue（写清这阶段要什么）→ 功能分支 → 让 Agent 实现
→ 亲自游玩测试 → git diff 检查 → 提交推送 → PR 合并
```

> 💡 八个阶段就是八个细粒度里程碑。**不要贪快合并阶段**——小步提交的好处是：哪一步玩坏了，就退回哪一步，前面的成果永远安全。这正是你学 Git 的意义。

---

## 阶段 1：搭建项目

🎯 **成功标志：** GitHub 存在 `snake-game` 私有仓库；本地有 `index.html`、`style.css`、`game.js`、`README.md`；第一次 commit 已推送。

**操作步骤**

1. 在 GitHub 创建私有仓库 `snake-game`（🔗 https://github.com/new）
2. 克隆并进入：

   ```bash
   cd ~/Desktop
   git clone 你的仓库地址
   cd snake-game
   ```

3. 在 WorkBuddy 中输入：

   ```text
   我要完成一个浏览器版贪吃蛇毕业项目。请先创建 index.html、style.css、game.js 和 README.md，只搭建项目结构，不要实现完整游戏。
   ```

4. 检查四个文件都在，然后：

   ```bash
   git add .
   git commit -m "搭建贪吃蛇项目结构"
   git push
   ```

🔍 **刚才发生了什么**
三个文件各司其职：`index.html` 是骨架（页面上有什么），`style.css` 是皮肤（长什么样），`game.js` 是大脑（怎么动）。网页开发的世界观就这三层。

## 阶段 2：显示游戏界面

🎯 **成功标志：** 打开 `index.html` 能同时看到 400×400 的游戏区域、「贪吃蛇」标题、0 分和开始按钮，页面无空白。

1. 创建 Issue：`显示贪吃蛇游戏基础界面`
2. 让 WorkBuddy 实现：

   ```text
   创建一个 400×400 像素的贪吃蛇游戏区域，显示"贪吃蛇"标题、当前分数和开始按钮。暂时不要让蛇移动。
   ```

3. 双击 `index.html` 亲眼验收。
   📸 截图 M6-2-1：基础界面
4. 固定动作收尾：

   ```bash
   git status
   git diff
   git add .
   git commit -m "完成游戏基础界面"
   git push
   ```

## 阶段 3：让蛇动起来

🎯 **成功标志：** 点击开始后，三个连续方块以固定速度向右移动至少 5 格，页面不卡死；本阶段走了分支 + PR。

1. 创建分支：

   ```bash
   git switch -c feature/snake-movement
   ```

2. 输入：

   ```text
   在现有页面中创建一条由三个方块组成的蛇。点击开始后，让蛇按照固定速度向右移动。暂时不实现食物和碰撞。
   ```

3. 多次刷新页面测试：蛇不忽快忽慢、页面不卡死。
   📸 截图 M6-3-1：移动中的蛇
4. 提交、推送、创建 PR、合并（流程忘了就回翻里程碑五任务 10）。

---

## 阶段 4：控制方向

🎯 **成功标志：** 方向键和 WASD 都能转向；快速连按不崩溃；向右移动时按左键**不会掉头**。

1. 创建 Issue 后输入：

   ```text
   给贪吃蛇增加键盘控制。方向键和 WASD 都能转向。蛇向右移动时不能直接向左掉头，其他方向同理。
   ```

2. 按测试清单逐项亲测：

   - [ ] 上、下、左、右有效
   - [ ] W、A、S、D 有效
   - [ ] 快速连续按键不异常
   - [ ] 不能直接反向掉头

3. 完成后检查、提交、推送、合并。

> 💡 「不能直接掉头」是这个游戏第一条**规则类需求**。写 Issue 时把这种边界规则写清楚，AI 的实现质量会高一大截。

## 阶段 5：食物与计分

🎯 **成功标志：** 连续吃到三个食物，分数依次 10、20、30；蛇每次长一格；食物从不出现在边界外或蛇身上。

1. 输入：

   ```text
   给游戏增加食物。食物必须随机出现在网格内，不能生成在蛇身上。蛇吃到食物后身体增加一格，分数加 10，并生成新的食物。
   ```

2. 亲自连吃三个，逐项核对上面四个成功标志。
   📸 截图 M6-5-1：得分 30、蛇身变长的画面

---

## 阶段 6：失败规则

🎯 **成功标志：** 亲自完成一次撞墙、一次撞自己——两次都立即停止移动并显示「游戏结束」和最终分数。

1. 输入：

   ```text
   增加游戏结束规则。蛇撞到墙或自己的身体时停止移动，页面显示"游戏结束"和最终分数。
   ```

2. **必须亲自制造两次失败**：主动撞墙一次；把蛇吃长后主动撞自己一次。

> ⚠️ 不要只看代码觉得「应该没问题」。测试的铁律：**没有亲手触发过的功能，等于没有这个功能。**

---

## 阶段 7：重新开始

🎯 **成功标志：** 连玩三轮，每次重开都恢复 0 分、三格蛇身、向右方向和新食物，且不需要刷新网页。

1. 输入：

   ```text
   增加重新开始功能。重新开始后分数归零，蛇恢复三个方块，方向恢复向右，重新生成食物，不刷新整个网页。
   ```

2. 连玩三轮，专门检查「上一轮的状态有没有残留」（分数没清零？蛇长度没恢复？）。

---

## 阶段 8：完善文档，异地复活验证

🎯 **成功标志：** README 五脏俱全；在全新文件夹 clone 后**不改一行代码**就能玩。

1. 让 WorkBuddy 完善 `README.md`，然后**亲自检查**是否包含：
   - [ ] 游戏介绍
   - [ ] 操作方法
   - [ ] 如何启动
   - [ ] 用了哪些文件
   - [ ] 已完成功能
   - [ ] 一张游戏截图
2. 最终推送：

   ```bash
   git status
   git log --oneline
   git push
   ```

3. 异地复活验证：

   ```bash
   cd ~/Desktop
   git clone 你的仓库地址 snake-game-final-check
   cd snake-game-final-check
   open index.html
   ```

   📸 截图 M6-8-1：新克隆目录中游戏正常运行

**重新下载后仍能玩，毕业项目才算真正完成。**

## 毕业寄语

> 你不需要先学会写出每一行代码，才有资格开发软件。你需要学会：**描述目标、拆分任务、运行程序、观察结果、发现错误、保存版本，并让 Agent 根据反馈继续修改。**
>
> 贪吃蛇不是终点。它是你第一次完整地指挥 AI 完成一个软件项目。
