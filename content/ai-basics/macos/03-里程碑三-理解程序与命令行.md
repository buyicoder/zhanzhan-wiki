## 3.1 LLM：一个输入话、输出话的黑箱

### 任务 1：建立黑箱模型

🎯 **成功标志：** 你能亲手画出「一段话 → LLM → 一段回答」，并指出哪边是输入、哪边是输出。

LLM 是大语言模型的简称。完全理解它很难，但直观理解很简单：

```text
输入一段话  ──→  ┌─────────┐  ──→  输出一段话
                │   LLM   │
                └─────────┘
```

它输出的话看起来有逻辑、像在回答你，简直和人一样。你平时用的豆包、DeepSeek，背后就是大语言模型。**先把它当黑箱**：不用管里面怎么运作，只看输入和输出。

**操作步骤**

1. 拿一张纸（或开个 Markdown 文件），画出上面的黑箱图。
2. 打开任意一个大模型对话工具，问三个不同类型的问题（一个知识题、一个创作题、一个判断题）。
3. 挑一条回答，找出一处「值得追问或核实」的地方，追问它。

🔍 **刚才发生了什么**
第 3 步是关键：LLM 的输出是概率生成的，**可能出错、可能编造**。「敢用 AI + 会核实」才是完整能力，缺一半都不行。

## 3.2 只做计算的程序

程序和 LLM 用同一个模型理解：

```text
输入  ──→  ┌─────────┐  ──→  输出
          │  程序    │
          └─────────┘
```

区别在于：程序的行为是确定的。加法程序，输入 1 和 2 一定输出 3；输入 100 和 200 一定输出 300。

### 任务 2：先把 Python 环境准备稳

🎯 **成功标志：** 新开一个终端后，能看到 Python 版本和程序位置，并成功输出 `python-ready`；电脑里没有因为反复安装出现一堆混乱的 Python。
⏱️ 预计用时：15–30 分钟

🔗 Python 官网：https://www.python.org/downloads/macos/

**操作步骤**

1. 在 WorkBuddy 中输入下面这段完整要求。先让它检测，不要一上来就安装：

   ```text
   我正在准备本课程的 Python 运行环境，我使用的是 macOS。请严格按下面的顺序帮我操作：
   1. 先检测 python3 --version 和 which python3，不要直接安装，也不要直接修改 PATH、.zshrc 或其它环境变量；
   2. 如果 python3 已经能正常运行，就不要重复安装；
   3. 如果没有可用的 Python，只使用 Python 官网的 macOS 安装器安装稳定版本，不要同时使用多种安装方式；
   4. 安装完成后提醒我完全关闭并重新打开终端和 WorkBuddy；
   5. 最后依次验证 Python 版本、程序位置，并运行 python3 -c "print('python-ready')"；
   6. 如果任何一步失败，先把检测结果和失败原因告诉我，不要继续创建程序，也不要反复安装。
   ```

2. 按 WorkBuddy 的提示完成检测或安装。需要下载时，只进入上方 Python 官网。
3. 如果刚刚安装了 Python，完全退出终端和 WorkBuddy，再重新打开。
4. 在新终端里亲自输入：

   ```bash
   python3 --version
   which python3
   python3 -c "print('python-ready')"
   ```

5. 确认第一行出现版本号，第二行出现一个实际路径，最后一行只输出 `python-ready`。
   📸 截图 M3-2-1：三个 Python 验证命令及完整输出

> ⚠️ 只在当前窗口临时成功不算准备完成。安装后重新打开终端仍然有效，才说明环境真的准备好了。不要为了“修好”而同时安装官网版、Homebrew 版和其它版本。

🔍 **刚才发生了什么**
Python 程序要由 Python 解释器运行。这里先确认解释器真实存在、终端能找到它，再开始写程序。**先把工具验收好，再用工具干活**，会比遇到报错后盲目修改环境变量可靠得多。

### 任务 3：创建并运行 add.py

🎯 **成功标志：** 用至少三组不同数字运行 `add.py`，每次都输出正确的和。
⏱️ 预计用时：15 分钟

**操作步骤**

1. 在 WorkBuddy 中输入：

   ```text
   我已经验证 python3 可以正常运行。请只在当前工作空间创建 add.py：程序启动后依次让我输入两个数字，然后输出它们的和。不要安装软件，不要修改环境变量，也不要创建其它文件。
   ```

2. 打开访达确认 `add.py` 出现在课程文件夹里。
3. 在课程文件夹中打开终端，输入：

   ```bash
   python3 add.py
   ```

4. 看到「请输入第一个数字：」→ 输入一个数，回车 → 输入第二个数，回车 → 看到结果。
   📸 截图 M3-3-1：终端中完整的一次加法运行过程
5. 重新输入 `python3 add.py`，换不同的数字**再跑两次**，验证每次都正确。

🔍 **刚才发生了什么**
你刚才「启动」了一个程序，给了它输入，收到了输出。程序跑完就退出了——所以要再算一次就要再启动一次。这个「启动 → 输入 → 输出 → 退出」的循环，就是一切程序的生命周期。

### 任务 4：举一反三——max.py 和其它运算

🎯 **成功标志：** `max.py` 用三组数据测试（最大数在开头、中间、结尾各一组），输出都正确；另外自选生成至少一个减法/乘法/除法程序并验证。

**操作步骤**

1. 在 WorkBuddy 中输入：

   ```text
   创建一个max.py文件，我输入五个数，它就会输出五个数中的最大值
   ```

2. 用 `python3 max.py` 运行，准备三组数据：
   - `9 1 2 3 4`（最大在开头）
   - `1 2 9 3 4`（最大在中间）
   - `1 2 3 4 9`（最大在结尾）
3. 自己出题：让 AI 再生成一个减法（或乘除法）程序，用两组数据验证。

> 💡 为什么要准备三组刁钻数据：这叫「边界测试」——程序在普通情况对，不代表在特殊情况也对。**会设计测试数据，是你验收 AI 工作成果的核心武器**，比会写代码更重要。

## 3.3 会产生副作用的程序

程序不仅能计算，还能**改变程序外面的世界**——创建文件、删除文件、发送消息。这类影响叫「副作用」（Side effect）。

### 任务 5：创建 create_folder.py

🎯 **成功标志：** 输入 `5`，文件夹里恰好新增 5 个子文件夹，同时程序输出完成提示。
⏱️ 预计用时：15 分钟

**操作步骤**

1. 在 WorkBuddy 中输入：

   ```text
   创建一个create_folder.py文件，输入一个数N，能创建N个文件夹
   ```

2. 终端运行：

   ```bash
   python3 create_folder.py
   ```

3. 输入 `5`，回车。
4. 打开访达，**数一数**是不是恰好多了 5 个文件夹。
   📸 截图 M3-5-1：访达中新出现的 5 个文件夹

5. 指认三件事（对自己说出来）：
   - 输入是什么？→ 数字 N
   - 输出是什么？→ 「已创建 N 个文件夹」这句话
   - 副作用是什么？→ 电脑上**真实出现了** N 个文件夹

### 任务 6（挑战）：批量删除程序

🎯 **成功标志：** 运行前能数到刚创建的文件夹；运行后它们全部消失，**其它文件毫发无损**。

**操作步骤**

1. 让 WorkBuddy 写一个批量删除刚才那批文件夹的程序。
2. 运行前，先确认它删的是哪些文件夹（问 AI：「这个程序会删除哪些东西？会不会误删别的？」）。
3. 运行，然后打开访达验证。

> ⚠️ 删除是最典型的「危险副作用」——错了无法撤销。**先看清会删什么，再运行。** 这个警觉性请保持一辈子。里程碑四的 Git 就是为这种时刻准备的后悔药。

🔍 **刚才发生了什么**
你现在理解了 Agent 的本质：**LLM（会说话的黑箱）+ 能产生副作用的程序 = 能操作电脑的 AI**。WorkBuddy 在里程碑一创建文件夹时，底层就是这么干的。

## 3.4 命令行：不用鼠标操作电脑

我们平时双击图标打开程序。但还有一种更强大的启动方式：**命令**。在鼠标发明之前，所有人都是用命令操作电脑的。

### 任务 7：用命令在文件夹之间移动

🎯 **成功标志：** `pwd` 的输出以课程文件夹名结尾；`ls` 能列出之前创建的 Python 文件。
⏱️ 预计用时：15 分钟

**操作步骤**

1. Cmd + 空格，输入「终端」，回车打开。你会看到一个等待输入的窗口。
2. 输入 `pwd`，回车。它会输出一串路径——`pwd` 的作用是告诉你「终端现在位于电脑的哪个文件夹」。
3. 输入 `ls`，回车。它会列出当前位置的文件和文件夹。
4. 进入课程文件夹：

   ```bash
   cd ~/Desktop/zhanzhan-ai-course
   ```

   > 💡 偷懒技巧：输入 `cd `（带空格）后，把文件夹从访达拖进终端，路径自动补全。
5. 再次输入 `pwd` 确认位置，再输入 `ls`，看到 `add.py`、`max.py` 就成功了。
   📸 截图 M3-7-1：pwd + ls 的完整输出

**先记住这三个命令，它们是你在电脑里的腿：**

| 命令 | 作用 | 记法 |
|------|------|------|
| `pwd` | 我在哪 | print working directory |
| `ls` | 这里有什么 | list |
| `cd` | 去某个地方 | change directory |

### 任务 8：理解「程序名 + 参数」

🎯 **成功标志：** 能说出 `python3 add.py` 里哪个是程序、哪个是参数。

**操作步骤**

1. 运行 `python3 add.py`，完成一次加法。
2. 回答：`python3` 是**被启动的程序**，`add.py` 是**交给它的参数**（告诉 python3 要运行哪个文件）。

🔍 **刚才发生了什么**
命令行的语法就这么一条：`程序名 参数1 参数2 ...`。终端本身不干活，它只是接收你的命令、启动对应的程序。学会这一条，你就看得懂 90% 的命令了。

### 任务 9：不打开访达，只用命令操作文件

🎯 **成功标志：** 最后 `ls` 能看到 `command-line-practice` 文件夹，里面有 `first-file.txt` 和 `renamed-file.txt`。

**操作步骤**（每输入一条就 `ls` 看一眼变化）

```bash
pwd                                             # 确认在课程文件夹
mkdir command-line-practice                     # 创建文件夹
cd command-line-practice                        # 进入它
touch first-file.txt                             # 创建空文件
cp first-file.txt second-file.txt                # 复制
mv second-file.txt renamed-file.txt              # 改名
ls                                              # 验收
```

📸 截图 M3-9-1：全套命令执行后的终端画面

**新增四条命令，它们是你在电脑里的手：**

| 命令 | 副作用 |
|------|--------|
| `mkdir` | 创建文件夹 |
| `touch` | 创建文件 |
| `cp` | 复制文件 |
| `mv` | 移动文件 / 改名 |

## 3.5 批处理脚本：把一串命令打包成一个程序

单条命令只能做一个动作。把多条命令按顺序写进一个文件，就成了「批处理脚本」——一次执行一整套操作。

### 任务 10：让 AI 编写 create_project.sh

🎯 **成功标志：** 运行脚本后自动生成一套完整的项目文件夹结构；删掉再跑一次，能原样重建。
⏱️ 预计用时：15 分钟

**操作步骤**

1. 在 WorkBuddy 中输入：

   ```text
   请帮我创建一个 create_project.sh 批处理脚本。运行脚本后，自动创建 my-first-ai-project 文件夹，并在里面创建 documents、code、images、output 四个文件夹和一个 README.md 文件。所有路径必须只使用小写英文、数字、短横线或下划线，不要使用中文路径、空格或括号。请先向我解释每一条命令，再创建脚本。
   ```

2. 在终端运行：

   ```bash
   bash create_project.sh
   ```

3. 验证是否出现这个结构：

   ```text
   my-first-ai-project
   ├── README.md
   ├── code
   ├── documents
   ├── images
   └── output
   ```

4. 打开脚本文件，找到里面的命令，至少指出两条并说出它们各创建了什么。
5. 删除 `my-first-ai-project` 文件夹，再运行一次脚本，确认能原样重建。
   📸 截图 M3-10-1：脚本两次运行后重建的目录结构

🔍 **刚才发生了什么——本章的大收束**
回忆黑箱模型：LLM 输入一段话、输出一段话。这段话可以是中文，**也可以是命令**。

> LLM 会输出命令 → 命令能启动程序 → 程序有副作用 → 副作用就是电脑操作。

这就是 AI 操作电脑的完整原理链。以前批处理脚本是程序员的专利；现在 AI 替你写脚本，你只需要**看得懂发生了什么、验得了收**。而这，正是你这一章练出来的能力。

🔬 **深入一层**
程序设计里有个概念叫「循环」：让计算机重复做一件事。要创建 100 个文件夹，鼠标点到手抽筋，循环只要三行。「重复的事情交给程序」是计算机思维的核心，也是你以后判断「这事该不该让 AI 写个脚本」的直觉来源。
